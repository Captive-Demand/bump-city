

# Bump City — Baby Shower Planning App

## Overview
Convert the BumpCity demo (a mobile-first baby shower planning app) into a production-ready React + Vite + Tailwind CSS + shadcn/ui app with Lovable Cloud backend.

**Note on framework**: Lovable does not support Next.js. This project uses **React + Vite**, which is the supported stack. You get Tailwind, shadcn/ui, and Lovable Cloud (Supabase-based backend with auth, database, storage) — covering the same capabilities.

---

## App Structure (from demo analysis)

The demo has these screens/sections:
- **Home/Dashboard** — welcome screen with baby name, countdown to due date, quick action cards
- **Registry** — gift registry with categories, items, claimed status
- **Guest List** — RSVP tracking with guest names and statuses
- **Games** — baby shower games/activities list
- **Profile/Settings** — user profile and event settings
- **Bottom Navigation** — tab bar for switching between sections

Design: Mobile-first (max 430px), pink/pastel color palette, rounded cards, playful typography.

---

## Technical Plan

### Step 1: Design System & Theme
- Update `index.css` CSS variables to match Bump City's pink/pastel palette
- Configure Tailwind with custom colors (soft pink, lavender, mint accents)
- Set up Google Fonts (matching the demo's typography)

### Step 2: Layout & Navigation
- Create a `MobileLayout` wrapper component (max-w-[430px], centered, min-h-screen)
- Build bottom tab navigation using shadcn Tabs or custom component with icons (Home, Registry, Guests, Games, Profile)
- Set up React Router routes for each tab

### Step 3: Pages (5 screens)
- **HomePage** — Baby name header, countdown timer, quick-action cards (shadcn Card)
- **RegistryPage** — Category filters, gift item cards with claim button (shadcn Card, Button, Badge)
- **GuestListPage** — Guest list with RSVP status badges, add guest form (shadcn Input, Badge, Dialog)
- **GamesPage** — Activity cards with descriptions and status
- **ProfilePage** — Event settings, user info (shadcn Form, Input, Switch)

### Step 4: Lovable Cloud Backend
- **Database tables**: `events`, `registry_items`, `guests`, `games`, `user_profiles`
- **Authentication**: Email/password sign-up and login
- **Row Level Security**: Users can only access their own event data
- **Storage**: For profile photos and registry item images

### Step 5: Connect Frontend to Backend
- Replace hardcoded data with Supabase queries (using `@tanstack/react-query`)
- Add auth flow (login/signup pages)
- Wire up CRUD operations (add/edit/delete registry items, guests, games)
- Real-time RSVP status updates

## Implementation Order
1. Theme + layout + bottom nav (visual shell)
2. All 5 pages with hardcoded data (match demo UI)
3. Enable Lovable Cloud + set up database tables
4. Auth flow (login/signup)
5. Connect pages to live data

This approach lets you see the UI immediately, then layer in the backend.
